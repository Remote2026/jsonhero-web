输入区 — textarea vs input

Part 2.5: 用 textarea 还是 input？

方案 A: Single-line input
PASTE JSON OR URL

问题

  • 大段 JSON 看不见
  • 需要滚动才能看完内容
  • 用户不确定"贴进去了吗"
✅ 方案 B: Multi-line textarea
PASTE JSON OR URL

优势

  • 大段 JSON 可见
  • 输入 URL 也完全 OK
  • 粘贴后用户能看到内容，有确认感
  • 等宽字体，JSON 结构一目了然

交互建议

  • 默认 4 行高度（~100px），足够展示中小型 JSON
  • 可拖拽调整高度（resize: vertical），满足大型 JSON
  • 等宽字体（JetBrains Mono），让 JSON 结构对齐可见
  • 提交按钮始终在下方固定，不随 textarea 高度变化位移
  • URL 输入作为次要入口：在 textarea 下方加一行"或输入 URL 链接"的小链接？还是合并？

关于 URL 输入的交互方式，你怎么看？

1

合并到 textarea

输入 URL 也贴在 textarea 里，自动识别

2

分离 URL 输入

textarea 下方加一个小的 URL 输入行 + 按钮

3

Tab 切换

两个 Tab："Paste JSON" / "URL"，切换显示 textarea 或 input
