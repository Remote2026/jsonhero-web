设计要点

  • 左操作 · 右预览 — 左侧占 ~40%，右侧占 ~60%
  • 信息层级清晰 — 标题 → 副标题 → 输入框 → 拖拽区 → 示例入口，从上到下逐级引导
  • 右侧深色代码区 — 模拟真实 JSON Hero 的查看体验，让用户立刻理解产品价值
  • 主题适配 — 首页目前强制深色模式 (root.tsx 中 forceDarkMode)，右侧预览区保持深色，左侧可亮可暗
  • 去掉 Hero 区视频（移到下方作为功能展示），腾出空间给核心操作
